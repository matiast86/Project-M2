About the Project

This project is part of Module 2 of the Full Stack Development program at Soy Henry. Throughout this module, I have worked on 14 assignments (Homeworks) that have allowed me to develop and consolidate my programming skills. Each assignment addresses different aspects and fundamental technologies for a Full Stack developer.

It consisted of creating the HTML and CSS structures of the site where information about film cards is shown.

A JavaScript file was also created to obtain the information on the films through an array of objects and display them on the main page.

I also made secondary pages with information about me and the project, along with a link to the IMDB site.

In this homework, jQuery and index.js were combined, and a function was implemented to make a GET request, which retrieves a JSON with information on the films.

Bootstrap was added to the project, and the card container was edited to organize it in a simpler way. Bootstrap elements were also added to the static pages.

The "front" app was modularized, and Webpack was added as a devDependency to compile all the modules into a single "bundle.js," which was injected into the index.html.

To practice TDD testing with Jest, I developed different tests for a class called "carritoCompras" which included various methods. First, unit tests were created, followed by the development of the class methods based on these tests.

The Axios library was installed, and potential outcomes were handled properly using async/await and try/catch blocks.

Nodemon was installed as a devDependency and Express as a regular dependency. A folder structure was created to work with, including the "src" folder and inside it the "controllers", "services", and "routes" folders. The modules "index.js" and "server.js" were also created in their respective folders. The package.json was configured to run "nodemon" with "start".

Express was then required, and the "app" object was created in server.js and exported to index.js, which executes the "listen" method with its corresponding arguments.

Finally, the necessary elements were implemented to wait for a GET request and, when received, execute the corresponding controller.

The dependencies "morgan" and "cors" were installed. The necessary configurations were made to route the requests through morgan, cors, and json middlewares. The "services" module was implemented, and the Movies class was developed to create the array of movies. The service then returns the array with the original films from the previous server.

In an Excel sheet, a table with three movies was created. Each column represents a property, and different tables were included to represent different "entities". These entities were then related to the movie table.

An account was created in MongoDB, and a cluster for the "movies" database with a collection named "movies" was set up. The three movies were copied and loaded as documents in the collection with all their properties. MongoDB Compass was also downloaded.

The dependency "mongoose" was installed, and the connection to the database was set up. A schema for movies was also created, exported, and required in the services module using the "find()" method to get the movies, ensuring the response from the controller includes the actual data from the database.

A POST path was created along with the necessary functions to create new films in the database. An HTML page was set up with a form for inputs. A custom middleware was also created to validate all inputs from the form.

A "clear fields" button was added to the form along with functions to obtain the information from the form.

This was the final project. The necessary routes and logic were implemented with Express and integrated with the POST method on the front-end app. When the form information is submitted, the movie is created in the database. Now, new films can be created and displayed on the main page.
